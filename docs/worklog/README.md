# docs/worklog/ — the session paper trail

A short, dated entry for each meaningful work session — by an agent or a human. The worklog is
how work survives the context window: what shipped, what was decided, and what's still open, so
the next session (or the next agent) starts with a clean handoff instead of re-deriving it.

## Rules

- **One entry per session**, named `YYYY-MM-DD-<short-slug>.md`.
- **Write it at the end of the session** (or the end of each phase for large work).
- Keep it tight: verb + what + where. Link the feature and research docs it relates to.
